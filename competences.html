Network Redesign of the Antarctica Station

Semester 2

This university project involved transitioning from a Wi-Fi to a wired network. It was carried out in a group of four and divided into two parts. The first part consisted of an in-depth study of the initial situation in order to propose a solution that included a hardware architecture, a troubleshooting diagram, a cabling plan, recommended equipment types, and a detailed cost estimate. We also had to configure NAT (both static and dynamic PAT) to allow each machine to access the Internet and enable external machines to access Apache web pages. The second part involved implementing the proposed solution from the first part by configuring various services such as DNS, Active Directory (AD), DHCP, and Apache, as well as setting up active network equipment according to the troubleshooting diagram. During the project, I was responsible for DNS, NAT, the hardware architecture, the troubleshooting diagram, the cost estimate, Active Directory (AD) in collaboration with a teammate, and the configuration of active network devices.

Amplifier Design

Semester 3

This project involves the design of an amplifier operating at a specific frequency of 1.7 GHz using the ADS software. The objective is to implement the various stages of RF design: DC biasing of a transistor, simulation of S-parameters, replacing ideal components with real ones, and creating input and output impedance matching networks using the Smith Chart. Successive simulations are used to analyze the circuit’s behavior, particularly in terms of gain, matching, and isolation. A final optimization phase allows for fine-tuning of components to meet strict performance criteria (S11 and S22 < –20 dB). This project develops practical skills in high-frequency circuit design and in the use of professional simulation tools.

Network and Service Security

Semester 4

We designed and configured a secure network infrastructure linking two simulated company sites. Developed logical architecture and cabling plan with OSPF dynamic routing, network segmentation (clients/servers, public/private), and key services (HTTP, SSH, FTP). Deployed centralized DNS, DHCP for private clients, and IPSec tunnels (excluding Stormshield for encryption). Configured routers/switches with ACLs; Stormshield firewall used for captive portal only. Delivered detailed report and weekly log. Strengthened skills in network security, Cisco configuration, project management, and technical documentation

Client-Server Application

Semester 3

We develop a client-server application in Python using the UDP protocol to manage a race in real time through a graphical interface. Before the race starts, the user can define the number of checkpoints, add or remove runners, and then launch the race. As a runner passes through a checkpoint, their bib number is entered, the passage time is automatically recorded and sent to the server. At the end of the race, the server returns the final rankings and the total time taken by each runner.

Secure Multi-site Computer Network

Semester 3

We designed and deployed a complete network infrastructure across two interconnected company sites. This team-based project aimed to strengthen our skills in network architecture, dynamic routing (OSPF), configuration of CISCO equipment , and the implementation of various network services (DHCP, FTP, SSH, Web, RADIUS, QoS, etc.). Each site included three subnets: a public network for servers, a public network for clients, and a private network with access control and dynamic IP address assignment. We also implemented security policies, clear traffic segmentation, bandwidth reservation to ensure video server quality of service, and centralized authentication using RADIUS.
